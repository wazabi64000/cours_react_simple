0. La CLI : ton terminal est ton pote
La CLI c’est le petit carré noir où tu tapes des ordres et ton ordi obéit direct. Au lieu de cliquer comme un noob pendant 10 minutes, tu tapes 4 lignes et ton projet complet apparaît en 15 secondes. TOUS les vrais devs bossent comme ça.

La CLI = ligne de commande. C’est juste un terminal où tu tapes du texte et ton ordi obéit. C’est 100× plus rapide que cliquer partout.

Pourquoi on l’utilise ? Parce que c’est le seul moyen de lancer les outils modernes (Vite, npm, git…).

Ouvre ton terminal et tape ça :

npm create vite@latest blagues-en-folie -- --template react
cd blagues-en-folie
npm install
npm run dev

1. Vite : le Flash Gordon du dev
Vite démarre ton projet en 0,3 seconde alors que les vieux outils (Create React App) te font poireauter 20-30 secondes à chaque fois. En dev il compile rien, il laisse le navigateur charger les fichiers direct → c’est INSTANTANÉ. En prod il te sort un build propre et rapide. En 2025, si t’utilises pas Vite, t’es un dinosaure.

2. React en 3 phrases
React c’est le cerveau magique qui fait que quand tu changes une donnée (un compteur, une liste), l’écran se met à jour TOUT SEUL sans que t’aies à rafraîchir la page ou à toucher au HTML comme un plouc. Tu écris du HTML dans du JavaScript (c’est du JSX), tu découpes tout en petits morceaux réutilisables (les composants), et React fait tout le sale boulot pour toi. C’est utilisé par Facebook, Netflix, Instagram… donc ouais, c’est LA base.

React = bibliothèque pour faire des UI qui se mettent à jour toutes seules quand les données changent.

Tu codes des composants (petites briques) que tu réutilises partout.

Et grâce aux hooks, tu fais tout en fonctions, sans classes chiantes.

3. Composants : les Lego de ton app
Un composant c’est juste une fonction qui crache du HTML. Tu le fais une fois dans un fichier, tu l’importes, et tu l’utilises comme une balise normale <Blague />. Tu changes le composant à un endroit → il change PARTOUT. Finie la galère du copier-coller à la con. C’est propre, c’est clair, c’est la vie.

4. Props : les cadeaux entre composants
Les props c’est quand le composant papa file des trucs au composant gamin : « tiens, prends cette question et cette réponse ». Le papa décide ce qu’il envoie (<Blague question="..." reponse="..." />), le gamin reçoit ça dans les paramètres de sa fonction ({ question, reponse }) et il affiche. Le gamin peut lire mais il a pas le droit de modifier → c’est du sens unique, sinon c’est le bordel total. 
// src/components/Blague.jsx
function Blague({ question, reponse }) {
  return (
<div class="blague">
<h3>{question}</h3>
<p><strong>{reponse}</strong></p>
</div>
  );
}
export default Blague;
// src/App.jsx
<Blague
  question="Pourquoi JavaScript est-il triste ?"
  reponse="Parce qu’il n’a pas de class !"

5. useState : la mémoire du composant
useState c’est une fonction que tu appelles dans ton composant pour créer une variable spéciale.

Cette variable spéciale garde sa valeur même quand tu cliques sur un bouton ou que tu fais autre chose.

Quand tu changes cette variable avec la fonction qui vient avec, React rafraîchit automatiquement l’écran et affiche la nouvelle valeur.

Sans useState → si tu changes une variable normale, rien ne change à l’écran.
Avec useState → tu changes la variable → l’écran se met à jour tout seul.

Tu écris :
const [nombre, setNombre] = useState(0);
- nombre = la valeur actuelle
- setNombre = la fonction pour la changer

Quand tu fais setNombre(5), React re-rend ton composant et affiche 5 à la place de 0. 
import { useState } from 'react';
const blagues = [
  "Pourquoi les développeurs détestent la nature ? Trop de bugs.",
  "Comment appelle-t-on un boomerang qui ne revient pas ? Un bâton.",
  "Pourquoi les oiseaux ne jouent pas à cache-cache ? Parce qu’ils ont toujours le bec qui dépasse."
];
function Generateur() {
  const [index, setIndex] = useState(0);
  return (
<div style={{textAlign:'center', margin:'50px'}}>
<h2>{blagues[index]}</h2>
<button onClick={() => setIndex((i) => (i + 1) % blagues.length)}
              style={{fontSize:'2rem', padding:'15px 30px'}}>
        Nouvelle blague !
</button>
</div>
  );
}

6. useContext : le sac partagé de toute l’app
useContext c’est une façon de partager des données entre tous les composants sans devoir les passer un par un.

Tu crées un sac (on l’appelle un Context) tout en haut de ton application.
Tu mets dedans ce que tu veux partager (par exemple la liste des blagues favorites).

Ensuite, n’importe quel composant, même très loin dans l’arbre, peut ouvrir ce sac et prendre ce qu’il y a dedans.

Le mot « Provider » c’est juste le composant qui porte le sac et le donne à tout le monde.
Tu l’entoures autour de ton <App /> dans main.jsx.

Résultat : tous les composants peuvent accéder aux mêmes données sans galère. 
// src/context/FavoritesContext.jsx
import { createContext, useContext, useState } from 'react';
const FavoritesContext = createContext();
export function FavoritesProvider({ children }) {
  const [favorites, setFavorites] = useState([]);
  const add = (blague) => setFavorites(prev => [...prev, blague]);
  const remove = (blague) => setFavorites(prev => prev.filter(b => b !== blague));
  return (
<FavoritesContext.Provider value={{ favorites, add, remove }}>
      {children}
</FavoritesContext.Provider>
  );
}
export const useFavorites = () => useContext(FavoritesContext);
// Dans main.jsx
import { FavoritesProvider } from './context/FavoritesContext';
ReactDOM.createRoot(document.getElementById('root')).render(
<FavoritesProvider>
<App />
</FavoritesProvider>
);

7. React Router : plusieurs pages sans recharger
Le routing c’est juste dire : « quand l’URL est /, montre la page d’accueil, quand c’est /favorites, montre la liste des favorites ». Pas de rechargement complet de la page comme dans les années 2000. Tout reste fluide, l’état (tes favorites, ton compteur) est conservé. C’est une vraie application moderne (on appelle ça une SPA = Single Page Application).

8. useRef : attraper un élément DOM
useRef te permet de prendre un élément HTML (comme un <input>) et de le manipuler directement.

Tu crées une référence avec useRef().
Tu la colles sur l’élément avec ref={maRef}
Ensuite tu peux accéder à cet élément avec maRef.current

Par exemple :
- Lire ce que l’utilisateur a tapé : maRef.current.value
- Vider le champ : maRef.current.value = ''
- Remettre le curseur dedans : maRef.current.focus()

Contrairement à useState, changer une ref ne rafraîchit PAS l’écran → c’est parfait pour ces petites actions. 
import { useRef } from 'react';
function AjoutBlague() {
  const inputRef = useRef();
  const ajouter = () => {
    const nouvelle = inputRef.current.value;
    if (nouvelle) {
      alert("Blague ajoutée : " + nouvelle);
      inputRef.current.value = '';
      inputRef.current.focus();
    }
  };
  return (
<div style={{margin:'50px', textAlign:'center'}}>
<input ref={inputRef} placeholder="Ta blague pourrie ici" style={{fontSize:'1.5rem', padding:'10px'}} />
<button onClick={ajouter} style={{fontSize:'1.5rem', padding:'10px 20px'}}>Envoyer</button>
</div>
  );
}
