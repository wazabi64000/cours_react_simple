0. La CLI : ton terminal est ton pote
La CLI (Command Line Interface) c’est le terminal noir où tu tapes des commandes comme un vrai hacker. Au lieu de cliquer 50 fois comme un noob, tu tapes 4 lignes et ton projet entier apparaît en 10 secondes. Tout le monde sérieux utilise ça parce que c’est 100 fois plus rapide et puissant que les interfaces graphiques de merde.

La CLI = ligne de commande. C’est juste un terminal où tu tapes du texte et ton ordi obéit. C’est 100× plus rapide que cliquer partout.

Pourquoi on l’utilise ? Parce que c’est le seul moyen de lancer les outils modernes (Vite, npm, git…).

Ouvre ton terminal et tape ça :

npm create vite@latest blagues-en-folie -- --template react
cd blagues-en-folie
npm install
npm run dev

1. Vite : le Flash Gordon du dev
Vite c’est le mec qui te fait démarrer ton projet en 0,3 seconde alors que les vieux outils (Create React App) te font attendre 15-30 secondes à chaque fois comme un con devant un feu rouge. En développement il ne compile rien à l’avance, il laisse le navigateur faire le taf → c’est instantané. En production il te sort un build propre et rapide. Bref, si tu utilises pas Vite en 2025, t’es un dinosaure.

Vite démarre ton projet en 0,2 seconde (contre 10-20s avec Create React App). Il ne bundle rien en dev → c’est instantané.

En prod il utilise Rollup et sort un code ultra-optimisé.

Bref : Vite = plaisir immédiat, pas d’attente comme un vieux modem 56k.

2. React en 3 phrases
React c’est la bibliothèque qui te permet de construire des sites où tout se met à jour tout seul quand une donnée change, sans que t’aies à rafraîchir la page ou à toucher au DOM à la main comme un sauvage. Tu écris du HTML dans du JavaScript (JSX), tu découpes tout en petits morceaux réutilisables (composants), et React fait le reste. C’est utilisé par Facebook, Netflix, Instagram… donc ouais, c’est la base.

React = bibliothèque pour faire des UI qui se mettent à jour toutes seules quand les données changent.

Tu codes des composants (petites briques) que tu réutilises partout.

Et grâce aux hooks, tu fais tout en fonctions, sans classes chiantes.

3. Composants : les Lego de ton app
Un composant c’est juste une fonction qui retourne du HTML (JSX). Tu le crées une fois dans un fichier, tu l’importes et tu l’utilises comme une balise perso <Blague />. C’est la fin du copier-coller débile : tu changes le composant à un endroit, il change partout. C’est propre, c’est clair, c’est la vie.

<p>Parce que sinon ils tombent dans le bateau !</p>

4. Props : les cadeaux entre composants
Les props c’est quand le composant parent envoie des données au composant enfant, comme passer une bière à ton pote. Le parent décide ce qu’il envoie (<Blague question="..." reponse="..." />), l’enfant reçoit dans les paramètres de sa fonction ({ question, reponse }) et les affiche. L’enfant ne peut PAS modifier les props, c’est du sens unique → ça évite le bordel total. 
// src/components/Blague.jsx
function Blague({ question, reponse }) {
  return (
<div class="blague">
<h3>{question}</h3>
<p><strong>{reponse}</strong></p>
</div>
  );
}
export default Blague;
// src/App.jsx
<Blague
  question="Pourquoi JavaScript est-il triste ?"
  reponse="Parce qu’il n’a pas de class !"
/>

5. useState : la mémoire courte du composant
useState c’est le seul moyen de dire à React : « cette variable-là, je veux qu’elle survive entre les re-renders et que tu rafraîchisses l’écran dès que je la change ». Sans useState, si tu changes une variable normale, rien ne se passe à l’écran. Avec useState, tu as [valeur, fonctionQuiChangeLaValeur], tu appelles la fonction → React re-rend tout proprement. 
import { useState } from 'react';
const blagues = [
  "Pourquoi les développeurs détestent la nature ? Trop de bugs.",
  "Comment appelle-t-on un boomerang qui ne revient pas ? Un bâton.",
  "Pourquoi les oiseaux ne jouent pas à cache-cache ? Parce qu’ils ont toujours le bec qui dépasse."
];
function Generateur() {
  const [index, setIndex] = useState(0);
  return (
<div style={{textAlign:'center', margin:'50px'}}>
<h2>{blagues[index]}</h2>
<button onClick={() => setIndex((i) => (i + 1) % blagues.length)}
              style={{fontSize:'2rem', padding:'15px 30px'}}>
        Nouvelle blague !
</button>
</div>
  );
}

6. useContext : le pot commun de favorites
useContext c’est le moyen d’avoir un « frigo global » dans ton app : tu crées un Provider tout en haut (dans main.jsx), tu y mets tes données (ici la liste des favorites), et n’importe quel composant, même 10 niveaux plus bas, peut dire « donne-moi les favorites » sans faire passer la donnée à travers 50 props comme un jeu du téléphone arabe à la con. C’est propre, c’est rapide, c’est obligatoire dès que t’as plus de 2-3 niveaux. 
// src/context/FavoritesContext.jsx
import { createContext, useContext, useState } from 'react';
const FavoritesContext = createContext();
export function FavoritesProvider({ children }) {
  const [favorites, setFavorites] = useState([]);
  const add = (blague) => setFavorites(prev => [...prev, blague]);
  const remove = (blague) => setFavorites(prev => prev.filter(b => b !== blague));
  return (
<FavoritesContext.Provider value={{ favorites, add, remove }}>
      {children}
</FavoritesContext.Provider>
  );
}
export const useFavorites = () => useContext(FavoritesContext);
// Dans main.jsx
import { FavoritesProvider } from './context/FavoritesContext';
ReactDOM.createRoot(document.getElementById('root')).render(
<FavoritesProvider>
<App />
</FavoritesProvider>
);

7. React Router : plusieurs pages sans recharger
Le routing avec React Router c’est juste dire : « quand l’URL est /, montre la page d’accueil, quand c’est /favorites, montre la liste des favorites ». Pas de rechargement de page complet comme dans les années 2000, tout reste fluide, l’état (tes favorites, ton compteur) est conservé, c’est une vraie application moderne (SPA = Single Page Application).

8. useRef : attraper un élément DOM
useRef te donne un crochet pour attraper directement un élément HTML (input, div, etc.) et faire des trucs avec sans passer par l’état. Exemple : lire la valeur d’un input au moment du clic, faire un focus automatique, mesurer une taille… Ça ne déclenche pas de re-render à chaque fois (contrairement à useState), donc c’est parfait pour les opérations « sales » sur le DOM. 
import { useRef } from 'react';
function AjoutBlague() {
  const inputRef = useRef();
  const ajouter = () => {
    const nouvelle = inputRef.current.value;
    if (nouvelle) {
      alert("Blague ajoutée : " + nouvelle);
      inputRef.current.value = '';
      inputRef.current.focus();
    }
  };
  return (
<div style={{margin:'50px', textAlign:'center'}}>
<input ref={inputRef} placeholder="Ta blague pourrie ici" style={{fontSize:'1.5rem', padding:'10px'}} />
<button onClick={ajouter} style={{fontSize:'1.5rem', padding:'10px 20px'}}>Envoyer</button>
</div>
  );
}
